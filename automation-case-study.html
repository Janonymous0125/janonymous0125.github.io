What this script covers

Built around practical administration steps that are easy to repeat.

  • Self-elevates to administrator mode before changing system settings
  • Creates a SuperAdmin account and adds it to the Administrators group
  • Applies optional time limits to a selected local user account
  • Updates the hosts file and creates a daily shutdown task

Project overview

What problem this was built to solve

Setting up Windows machines by hand often means repeating the same account, restriction, and system configuration steps over and over. That slows the process down and makes it easier to miss details or apply settings inconsistently.

This script packages those actions into a single workflow so the setup process is easier to review, easier to repeat, and less dependent on remembering each step manually. Instead of acting as a generic experiment, the project focuses on practical administration work with a clear operational goal.

The result is a stronger portfolio proof item because it shows scripting, repeatability, and system-level thinking in one place rather than only offering a raw file download.

Primary goal

Reduce repetitive Windows setup work by grouping common admin tasks into one guided PowerShell flow.

Tools used

PowerShell, Windows local user management, hosts file editing, and Task Scheduler automation.

Workflow

Key capabilities inside the script

Administrator guardrail

The script checks for administrator privileges first and relaunches itself with elevation when needed so the workflow starts with the required permission level.

Account setup

It creates a dedicated SuperAdmin user, prompts for a secure password, and adds that account to the local Administrators group as part of setup.

User control steps

It can apply default or custom logon time limits to a selected local user while blocking changes to highly privileged accounts.

Hosts file updates

The workflow appends blocked domains to the Windows hosts file and includes retry logic so file locking does not immediately break the process.

Scheduled shutdown

It registers a daily shutdown task at 10:30 PM to help enforce a repeatable end-of-day system policy.

Cleanup mode

If the SuperAdmin account already exists, the script switches into cleanup mode and attempts to undo its previous changes instead of running setup again.
